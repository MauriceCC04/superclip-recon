# Qualitative results

This folder is reserved for qualitative artifacts used in the report, for example:

- selected success cases
- selected failure cases
- short per-example notes
- side-by-side comparisons of baseline vs reconstruction runs

Suggested convention:

- keep one markdown file per comparison or theme
- include the checkpoint / run name that produced each observation
- link back to the corresponding raw JSON result file when relevant

This folder is intentionally separate from raw result JSONs and automated plots.
